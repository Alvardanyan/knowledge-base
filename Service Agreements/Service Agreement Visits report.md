# Service Agreement Visits report

## Overview

Use the Service Agreement Visits report to review all visits in all statuses. The report is based on the Service Agreements visits report template, and it helps you monitor and manage all SA visits for a selected visit window date range, including details such as customer and location records, Service Agreements, and visit statuses.

## Who uses this feature

- Administrators, estimators, project managers, and operations
managers
- Primarily benefits Commercial Service and Replacement business
types
- Applies to all trades

## Things to know

- If you're unsure which columns you want to include in your report, use the
Reporting Dictionary to search for appropriate data fields.

## Benefits of using this report

- Streamlined tracking of all visits: Monitor every visit in a single place,
regardless of status. This allows you to identify dismissed or upcoming
visits and ensure that all visits are fulfilled as promised. By having all
customer, location, and service agreement details in one report, you can
manage follow-ups more effectively and reduce operational errors.

## Run report

The Service Agreement Visits report is a built-in report that's available to run
without any required setup.

### Locate the report

1. Go to the navigation bar and click Reports > All Reports.
2. In the search field, enter Service Agreement Visits, then click the report
card.

### Filter the report

This report has the following filters automatically set:

- From the Business Unit dropdown, select the Business Units (BUs) you want to report on. By default, the report runs for all BUs.
- From the Service Agreement Status dropdown, select the status of SAs.
The options include Activated, Canceled, and Expired.
- From the Visit Status dropdown, select the visit status. By default, the
report runs for all visit statuses.
- Click Visit Window From - Visit Window To to set date filters. You can:

- Use the calendar to select a specific date range and click Apply.
- Use the left menu to select a date range:
- Today: From 12:00 AM to the current time.
- Yesterday: Previous day, from 12:00 AM to 11:59 PM.
- Last 7 Days: Previous seven days, including the current date.
- Last 14 Days: Previous 14 days, including the current date.
- Last 30 Days: Previous 30 days, including the current date.
- Month to Date: From the first of the current month to the current day.
- Last Month: Entire month before the current month. For example, if
the date is February 5, the entire month of January.
- Last 90 Days: Previous 90 days, including the current date.
- This Quarter: Current quarter, including future dates. For example, if
the date is April 25, from April 1 - June 30.
- Last Quarter: Entire quarter before the current quarter. For example,
if the date is April 25, from January 1 - March 31.
- Quarter to Date: From the first day of the current quarter to the
current day.
- Year to Date: From January 1 of the current year to the current day.
- Last 365 Days: Previous 365 days, including the current date.
- Last Year: January 1 - December 31 of the previous calendar year.
- When finished, click Run Report.

## Available KPIs

Click Edit Report to add the following KPIs to the report.

- Business Unit: BU assigned to the SA.
- Customer PO #: Purchase Order number on the SA.
- Item Type: Type of the item used in a visit.
- Item Name: Name of the item.
- Item Unit Cost: Cost of the item.
- Primary Vendor: Primary vendor of the item.
- Invoice Item ID: Item ID on a visit invoice.
- Purchase Order #: Purchase order number.
- Item Code: Code of the item.
- Item Qty: Number of items used.
- Item Unit Price: Price of the item.
- Equipment Name: Name of the equipment.
- Procurement Status: Status of the procurement item.
- Job ID: ID of the job that the visit was booked for.
- Job Scheduled Date: Date of the job that was scheduled for the visit.
- Job Tags: Tags on the job.
- Job Number: Number of the job scheduled for the visit.
- Job Completed Date: Date of completion of the job.
